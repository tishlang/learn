Reading the docs

The most important skill of an experienced programmer isn't memorizing functions — it's finding and using documentation efficiently.

Tish's primary references

  • LANGUAGE.md — the canonical syntax + semantics spec.
  • tishlang.com/docs — narrative tutorials and API reference.
  • /builtins in this app's tish-deep track — searchable list of console / Math / Array / String / Object methods with examples.

How to search

When you want to know "how do I do X?":

1. Search the docs for the closest verb (split, format, parse). 2. Read the examples before the prose. One example is worth a paragraph. 3. Try it. Open a :::tryit block (or the playground) and verify your understanding.
