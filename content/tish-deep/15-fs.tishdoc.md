The `fs` feature exposes a small POSIX-style API.

```tish
import { readFile, writeFile, fileExists, readDir, mkdir } from "fs"

const text = await readFile("notes/ideas.md")
await writeFile("notes/done.md", "shipped")
console.log(fileExists("notes/done.md"))    // true
const list = await readDir("notes")           // string[]
await mkdir("notes/2026")
```

`readFile` returns either a string (success) or `{ error: "..." }` on failure — **it doesn't throw**. The pattern:

```tish
fn isFileError(v) {
  if (v === null) { return true }
  if (typeof v === "object" && "error" in v) { return true }
  return false
}

const r = await readFile(path)
if (isFileError(r)) { handleMissing(); return }
const text = r
```

## Atomic writes

`writeFile` is a one-shot replace. For atomic semantics across reads and writes, write to a temp path and rename:

```tish
await writeFile("data.json.tmp", encoded)
// then atomically rename via fs (when implementation lands) or accept overwrite races
```

## Directories

`readDir` returns immediate children only — no recursion. To walk a tree:

```tish
async fn walk(dir, out) {
  const entries = await readDir(dir)
  let i = 0
  while (i < entries.length) {
    const path = dir + "/" + entries[i]
    if (path.indexOf(".") < 0) { await walk(path, out) }
    else { out.push(path) }
    i = i + 1
  }
}
```

(The dot-test heuristic is rough; a real `stat` API is on the roadmap.)
